### Проверить RLS политики

```bash

FROM pg_policies 
WHERE tablename = 'monthly_analytics'
ORDER BY policyname;
"
```

Должно быть **7 политик**:
1. ✅ Admins can delete monthly analytics
2. ✅ Admins can insert monthly analytics for any user ← **ВАЖНАЯ**

4. ✅ Admins can update monthly analytics
5. ✅ Service role can manage monthly analytics
6. ✅ Users can insert their own monthly analytics

### Проблема: "Policy already exists"

Сначала удалите таблицу полностью:
```bash
psql postgresql://postgres:postgres@127.0.0.1:54322/postgres -c "
DROP TABLE IF EXISTS public.monthly_analytics CASCADE;
"
```

Затем примените миграции заново.

### Проблема: После миграций все равно ошибка 42501

Проверьте, что ваш пользователь действительно админ:
```bash
psql postgresql://postgres:postgres@127.0.0.1:54322/postgres -c "
SELECT 
  u.email,
  u.sso_uid,
  p.user_id,
  p.role,
  p.active
FROM users u
LEFT JOIN psychologists p ON p.user_id = u.sso_uid
WHERE u.email = 'ваш_email@example.com';
"
```

Должно быть:
- ✅ `u.sso_uid` = `p.user_id`
- ✅ `p.role` = 'admin'
- ✅ `p.active` = true
