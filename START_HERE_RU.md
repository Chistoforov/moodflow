## 📋 Чек-лист перед запуском

### 1. Применить миграцию БД ⏱️ 2 минуты

Откройте Supabase Dashboard → SQL Editor и выполните:
```bash
supabase/migrations/024_create_monthly_analytics.sql
```

**Или** следуйте инструкции в `APPLY_MIGRATION_024.md`

### 2. Проверить переменные окружения ⏱️ 1 минута

Убедитесь, что в `.env.local` и Vercel есть:
```bash
PERPLEXITY_API_KEY=pplx-...
CRON_SECRET=...
```

### 3. Деплой на Vercel ⏱️ 5 минут

```bash
git add .
git commit -m "feat: add monthly analytics"
git push origin main
```

### 4. Тестирование ⏱️ 3 минуты

1. Создайте 3+ записи в дневнике
2. Откройте страницу Календарь
3. Нажмите кнопку "Сгенерировать"
4. Через 10-30 секунд увидите аналитику

## ❓ FAQ

**Q: Сколько записей нужно для генерации?**
A: Минимум 3 записи в месяце.

**Q: Как часто генерируется аналитика?**
A: Каждые 7 дней + последний день месяца.

**Q: Можно ли удалить старую аналитику?**
A: Да, но финальная аналитика предназначена для истории.

**Q: Сколько стоит Perplexity API?**
A: Зависит от тарифа. Обычно ~$0.01-0.05 за запрос.

**Q: Что делать, если аналитика не генерируется?**
A: Проверьте логи Vercel, PERPLEXITY_API_KEY, и количество записей.
