# 🔄 Пересоздание анализа настроения

## Что изменилось

Теперь вы можете **принудительно пересоздать анализ**, даже если он уже существует. Это позволяет повторно отправить запрос в Perplexity для получения свежего анализа.

## Как пересоздать анализ

### Шаг 1: Откройте админ-панель
Перейдите на страницу `/admin/users`

### Шаг 2: Выберите пользователя
Нажмите кнопку **"📊 Анализ"** у нужного пользователя

### Шаг 3: Включите опцию пересоздания
В появившейся модалке вы увидите чекбокс:

```
☑️ Пересоздать анализ
   Если анализ уже существует, он будет удалён и создан заново
```

### Шаг 4: Запустите анализ
Нажмите кнопку **"Запустить"**

## Как это работает

### Без чекбокса (по умолчанию)
- ✅ Если анализ **НЕ существует** → создает новый анализ
- ℹ️ Если анализ **уже существует** → возвращает существующий (экономия запросов)

### С включенным чекбоксом
- 🗑️ Удаляет существующий анализ
- 🚀 Отправляет новый запрос в Perplexity
- ✅ Создает свежий анализ

## Результат

После завершения вы увидите одно из сообщений:
- ✅ **"Анализ успешно создан! Проанализировано X дней и Y записей"** - новый анализ создан
- ℹ️ **"Анализ для этого периода уже существует"** - анализ уже был
- ❌ **"Недостаточно записей для анализа (минимум 3)"** - мало данных

## Когда использовать пересоздание

1. 📝 **Добавили новые записи** - пользователь добавил записи после анализа
2. 🔄 **Нужен свежий анализ** - хотите получить новый вариант рекомендаций
3. 🐛 **Был сбой** - предыдущий анализ создался с ошибкой

## Внимание

⚠️ **Пересоздание удаляет старый анализ навсегда!**
⚠️ **Расходует кредиты Perplexity API**

Используйте эту функцию осознанно.
