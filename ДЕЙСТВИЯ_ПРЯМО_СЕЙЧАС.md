## 🔴 ВАЖНО: Проверьте ключи!

Если ваши ключи `SUPABASE_SERVICE_ROLE_KEY` и `NEXT_PUBLIC_SUPABASE_ANON_KEY` **одинаковые** - это ошибка!

👉 **Откройте файл `ИСПРАВИТЬ_КЛЮЧИ_СЕЙЧАС.md`** для быстрого исправления (3 минуты)

В Supabase есть ДВА разных ключа:
- **anon public** - для клиента (виден сразу)
- **service_role** - для сервера (под кнопкой "Reveal")

Они должны быть РАЗНЫМИ!

## 🔥 СЛЕДУЮЩИЕ ШАГИ

### ШАГ 1: Дождитесь завершения деплоя (2-3 минуты)

1. Откройте https://vercel.com
2. Перейдите в ваш проект
3. Откройте вкладку **Deployments**
4. Дождитесь, пока последний деплой завершится (статус "Ready")

### ШАГ 2: Добавьте переменные окружения ⭐ КРИТИЧНО

**Без этого шага приложение не заработает!**

Выберите один из вариантов:

#### 🟢 ВАРИАНТ A: Быстрое исправление (2 минуты)

Откройте файл **`БЫСТРОЕ_ИСПРАВЛЕНИЕ.md`** и следуйте 3 простым шагам.

Кратко:
1. Найдите Supabase ключи (URL, anon key, service_role key)
2. Добавьте их на Vercel → Settings → Environment Variables
3. Нажмите Redeploy

#### 🔵 ВАРИАНТ B: Автоматическая установка (5 минут)

Если у вас есть файл `.env.local`:

```bash
# 1. Установите Vercel CLI
npm i -g vercel

# 2. Авторизуйтесь
vercel login

# 3. Свяжите проект
vercel link

# 4. Запустите скрипт
./fix-vercel-env.sh
```

#### 🟣 ВАРИАНТ C: Полная настройка (10 минут)

Откройте файл **`FIX_MIDDLEWARE_NOW.md`** для подробных инструкций.

### ШАГ 3: Проверьте сайт

После добавления переменных и redeploy:

1. Откройте ваш сайт: https://moodflow-six.vercel.app
2. ✅ Если сайт открывается - отлично, все работает!
3. ❌ Если ошибка осталась - смотрите раздел "Если не помогло" ниже

## 📋 Минимальные переменные для запуска

Чтобы сайт заработал, добавьте **как минимум 3 переменные**:

1. `NEXT_PUBLIC_SUPABASE_URL` - URL вашего Supabase проекта
2. `NEXT_PUBLIC_SUPABASE_ANON_KEY` - публичный ключ Supabase
3. `SUPABASE_SERVICE_ROLE_KEY` - серверный ключ Supabase

**Где найти:** Supabase → Project Settings → API

## 🆘 Если не помогло

### 1. Проверьте Runtime Logs

Vercel → Deployments → кликните на последний деплой → Runtime Logs

Обновите сайт несколько раз и посмотрите на ошибки.

### 2. Убедитесь в правильности переменных

- ✅ Все 3 переменные добавлены
- ✅ Выбраны ВСЕ окружения (Production, Preview, Development)
- ✅ Нет опечаток в названиях
- ✅ Значения скопированы полностью

### 3. Если видите в логах "Missing Supabase environment variables"

Значит переменные не установлены или не применились. Решение:

1. Удалите все переменные окружения на Vercel
2. Добавьте их заново
3. Убедитесь, что выбрали Production окружение
4. Сделайте Redeploy
