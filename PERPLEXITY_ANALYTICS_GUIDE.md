# 📊 Руководство по Месячной Аналитике с Perplexity AI

## Обзор

Система месячной аналитики автоматически анализирует настроение пользователей с помощью Perplexity AI и предоставляет персонализированные рекомендации психологического характера.

## 🎯 Как это работает

### Механика накопления данных

1. **Еженедельная генерация**: Аналитика генерируется каждую неделю (каждые 7 дней)
2. **Накопительный принцип**: 
   - 1-я неделя: анализ за 7 дней
   - 2-я неделя: анализ за 14 дней (накопительно)
   - 3-я неделя: анализ за 21 день (накопительно)
   - 4-я неделя и далее: анализ за все дни месяца
3. **Финальный отчет**: В последний день месяца генерируется финальная аналитика, которая сохраняется навсегда

### Что анализируется

Для каждого дня передаются:
- 🌡️ Оценка настроения (1-5)
- 💭 Текстовая запись пользователя
- 🔖 Факторы, влияющие на настроение

## 📁 Структура базы данных

### Таблица `monthly_analytics`

```sql
CREATE TABLE monthly_analytics (
  id UUID PRIMARY KEY,
  user_id UUID NOT NULL,
  year INTEGER NOT NULL,
  month INTEGER NOT NULL,
  week_number INTEGER NOT NULL,      -- Номер недели (1, 2, 3, 4)
  days_analyzed INTEGER NOT NULL,     -- Количество проанализированных дней
  
  -- Структурированный анализ
  analysis_text TEXT NOT NULL,        -- Полный текст анализа
  general_impression TEXT,            -- Общее впечатление о периоде
  positive_trends TEXT,               -- Положительные тенденции
  decline_reasons TEXT,               -- Возможные причины спада
  recommendations TEXT,               -- Рекомендации и техники
  reflection_directions TEXT,         -- Направление для размышлений
  
  is_final BOOLEAN DEFAULT false,     -- true для последнего дня месяца
  status TEXT DEFAULT 'completed',
  created_at TIMESTAMP DEFAULT NOW()
);
```

## 🔄 API Endpoints

### 1. Генерация аналитики

**POST** `/api/analytics/generate`

```typescript
// Request
{
  year?: number,  // По умолчанию текущий год
  month?: number  // По умолчанию текущий месяц (1-12)
}

// Response
{
  success: true,
  analytics: { /* MonthlyAnalytics object */ },
  weekNumber: 2,
  daysAnalyzed: 14,
  isFinal: false
}
```

### 2. Получение аналитики

**GET** `/api/analytics?year=2024&month=11`

```typescript
// Response
{
  hasAnalytics: true,
  analytics: {
    id: "uuid",
    year: 2024,
    month: 11,
    week_number: 2,
    days_analyzed: 14,
    general_impression: "...",
    positive_trends: "...",
    decline_reasons: "...",
    recommendations: "...",
    reflection_directions: "...",
    is_final: false
  },
  isFinal: false
}
```

### 3. Cron Job (автоматический)

**GET** `/api/cron/monthly-analytics`

Запускается ежедневно в 22:00 UTC через Vercel Cron.

## 🤖 Perplexity Integration

### Промпт-инжиниринг

Система использует детализированный промпт, который:
- Представляет AI как профессионального психолога
- Запрашивает структурированный анализ по 5 категориям
- Включает контекст накопления (номер недели, количество дней)
- Форматирует записи с датами, настроением и факторами

## 📝 Логика работы

### Автоматическая генерация (Cron)

```
Каждый день в 22:00 UTC:
1. Проверить всех пользователей с активной подпиской
2. Для каждого пользователя:
   - Вычислить текущую неделю месяца
   - Проверить, нужно ли генерировать (каждые 7 дней или последний день)
   - Проверить, существует ли уже аналитика за эту неделю
   - Если нет - собрать все записи месяца до текущего дня
   - Отправить в Perplexity для анализа
   - Сохранить результат в БД
```

### Ручная генерация

Пользователь может вручную запросить генерацию аналитики:
- Кнопка "Сгенерировать" появляется, если аналитики еще нет
- Кнопка "Обновить" появляется для незавершенных месяцев
- Для финальной аналитики кнопка не показывается

## 🔒 Безопасность

### RLS Policies

```sql
-- Пользователи видят только свою аналитику
CREATE POLICY "Users can read their own monthly analytics"
  ON monthly_analytics FOR SELECT
  USING (auth.uid() = user_id);

-- Админы видят всю аналитику
CREATE POLICY "Admins can read all monthly analytics"
  ON monthly_analytics FOR SELECT
  USING (EXISTS (
    SELECT 1 FROM psychologists
    WHERE user_id = auth.uid() AND role = 'admin'
  ));
```

## 🎯 Особенности реализации

### Накопительный анализ

- Каждая новая неделя включает данные всех предыдущих дней месяца
- Perplexity получает информацию о номере недели в промпте
- Анализ учитывает динамику изменений в течение месяца

### Финальная аналитика

- Генерируется в последний день месяца
- Помечается флагом `is_final = true`
- Не может быть обновлена (кнопка "Обновить" скрыта)
- Сохраняется навсегда для исторического просмотра

### Оптимизация

- SWR кеширование для быстрой загрузки
- Дедупликация запросов
- Автоматическая ревалидация при генерации
- Минимум 3 записи для генерации аналитики

## 🐛 Troubleshooting

### Аналитика не генерируется

1. Проверьте PERPLEXITY_API_KEY в переменных окружения
2. Убедитесь, что пользователь имеет активную подписку
3. Проверьте, есть ли хотя бы 3 записи за месяц
4. Проверьте логи Vercel Functions

### Cron не запускается

1. Проверьте настройку cron в vercel.json
2. Убедитесь, что проект деплоится на Vercel
3. Проверьте логи в Vercel Dashboard -> Cron Jobs

### Ошибки RLS

1. Проверьте, что миграция 024 применена
2. Проверьте, что пользователь аутентифицирован
3. Для админов проверьте запись в таблице psychologists
