# 🔴 ИСПРАВЛЕНИЕ ОШИБКИ С КЛЮЧАМИ

## Проблема
Если `SUPABASE_SERVICE_ROLE_KEY` и `NEXT_PUBLIC_SUPABASE_ANON_KEY` **одинаковые** - это ошибка!

Они должны быть РАЗНЫМИ ключами.

## ⚡ БЫСТРОЕ ИСПРАВЛЕНИЕ (3 минуты)

### ШАГ 1: Найдите правильные ключи в Supabase

1. Откройте https://supabase.com
2. Выберите ваш проект
3. **Settings** (⚙️) → **API**
4. Прокрутите до "Project API keys"

Вы увидите:

```
anon public
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOi... 
[Копировать всё]
```

И ниже:

```
service_role                                    [Reveal]
```

**НАЖМИТЕ "Reveal"**, чтобы увидеть второй ключ:

```
service_role
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOi... (ДРУГОЙ токен!)
[Копировать всё]
```

📋 **Скопируйте ОБА ключа** в отдельные заметки, чтобы не перепутать.

### ШАГ 2: Обновите ключи на Vercel

1. Откройте https://vercel.com
2. Ваш проект → **Settings** → **Environment Variables**

#### Найдите SUPABASE_SERVICE_ROLE_KEY:

3. Кликните на три точки (⋯) рядом с `SUPABASE_SERVICE_ROLE_KEY`
4. Выберите **Edit**
5. **Удалите старое значение**
6. **Вставьте service_role ключ** (тот, который был под кнопкой "Reveal")
7. Убедитесь, что выбраны ВСЕ окружения: ✅ Production ✅ Preview ✅ Development
8. Нажмите **Save**

#### Проверьте NEXT_PUBLIC_SUPABASE_ANON_KEY:

9. Кликните на три точки (⋯) рядом с `NEXT_PUBLIC_SUPABASE_ANON_KEY`
10. Выберите **Edit**
11. Убедитесь, что там **anon public ключ** (первый, который был виден сразу)
12. Если нет - замените на правильный
13. Нажмите **Save**

### ШАГ 3: Redeploy

1. Перейдите в **Deployments**
2. Кликните на последний деплой
3. Три точки (⋯) → **Redeploy**
4. Дождитесь завершения (2-3 минуты)

### ШАГ 4: Проверьте сайт

Откройте ваш сайт - ошибка 500 должна исчезнуть! ✅

## 🔍 Как понять, что ключи правильные?

### Визуальная проверка:

Оба ключа начинаются с `eyJhbG...` но:
- anon key - обычно **короче**
- service_role key - обычно **длиннее**

Полные токены **ДОЛЖНЫ быть разными**!

### Проверка через CLI:

```bash
# Запустите скрипт проверки
./check-vercel-keys.sh
```

Он покажет, какие переменные установлены.

## 🆘 Если все еще не работает

### 1. Проверьте Runtime Logs

На Vercel:
1. **Deployments** → кликните на последний деплой
2. Перейдите в **Runtime Logs**
3. Обновите ваш сайт несколько раз
4. Посмотрите на ошибки в логах

### 2. Проверьте логи через CLI

```bash
vercel logs --follow
```

Обновите сайт и посмотрите на ошибки.

### 3. Частые ошибки в логах:

#### Если видите "Invalid JWT"
- Ключи скопированы неполностью
- Есть лишние пробелы в начале/конце
- **Решение:** Удалите и добавьте заново, копируя через кнопку "Copy"

#### Если видите "Missing Supabase environment variables"
- Переменные не установлены для Production окружения
- **Решение:** При редактировании убедитесь, что выбран Production

#### Если видите "Failed to parse session"
- Возможно используется старая сессия
- **Решение:** Очистите cookies браузера и попробуйте снова

#### Если видите "Connection refused" или "Network error"
- Проблема с NEXT_PUBLIC_SUPABASE_URL
- **Решение:** Проверьте, что URL правильный (без "/" в конце)

## 💡 Про VITE_SUPABASE_ANON_KEY

Этот ключ вы видите в Supabase для Vite проектов.

В Next.js он **НЕ используется**. Игнорируйте его, не добавляйте на Vercel.
