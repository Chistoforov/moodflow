# 🔥 СРОЧНО: Исправить ошибку PGRST205

## Проблема
При попытке анализа пользователя получаете ошибку **500 - Failed to save analytics: PGRST205**

## Причина
Базе данных не хватает прав для админов создавать аналитику для других пользователей.

---

## ⚡ Быстрое решение (2 минуты)

### Шаг 1: Откройте Supabase
1. Перейдите на https://app.supabase.com
2. Выберите проект **MoodFlow**
3. Откройте **SQL Editor** в левом меню

### Шаг 2: Выполните этот SQL
Скопируйте и вставьте в SQL Editor:

```sql
-- Добавить права админам для создания аналитики
CREATE POLICY "Admins can insert monthly analytics for any user"
  ON public.monthly_analytics
  FOR INSERT
  WITH CHECK (
    EXISTS (
      SELECT 1 FROM public.psychologists
      WHERE user_id = auth.uid()
      AND role = 'admin'
      AND active = true
    )
  );

CREATE POLICY "Admins can update monthly analytics"
  ON public.monthly_analytics
  FOR UPDATE
  USING (
    EXISTS (
      SELECT 1 FROM public.psychologists
      WHERE user_id = auth.uid()
      AND role = 'admin'
      AND active = true
    )
  );

CREATE POLICY "Admins can delete monthly analytics"
  ON public.monthly_analytics
  FOR DELETE
  USING (
    EXISTS (
      SELECT 1 FROM public.psychologists
      WHERE user_id = auth.uid()
      AND role = 'admin'
      AND active = true
    )
  );
```

### Шаг 3: Запустите
Нажмите **Run** (или `Ctrl/Cmd + Enter`)

Должны увидеть зелёные галочки ✅✅✅

### Шаг 4: Проверьте
Попробуйте снова запустить анализ пользователя в админ-панели.

**Ошибка должна исчезнуть!** 🎉

---

## ✅ Проверка что всё работает

В SQL Editor выполните:

```sql
SELECT COUNT(*) FROM pg_policies WHERE tablename = 'monthly_analytics';
```

Должно вернуть **7** (если видите 4, значит нужно выполнить SQL выше).
