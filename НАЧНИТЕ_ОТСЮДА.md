Ваши ключи `SUPABASE_SERVICE_ROLE_KEY` и `NEXT_PUBLIC_SUPABASE_ANON_KEY` **одинаковые** - в этом причина ошибки 500!

В Supabase есть **ДВА РАЗНЫХ** ключа, а вы использовали один и тот же для обеих переменных.

## 🎯 КРАТКО: Что нужно сделать

### 1. В Supabase (найти правильные ключи)

**Supabase → Settings → API**

Найдите два ключа:
- ✅ **anon public** (виден сразу) - скопируйте в блокнот
- ✅ **service_role** (нажмите "Reveal") - скопируйте в блокнот

⚠️ Это РАЗНЫЕ ключи!

### 2. В Vercel (обновить)

**Vercel → Settings → Environment Variables**

- Найдите `NEXT_PUBLIC_SUPABASE_ANON_KEY`
  - Edit → вставьте **anon public** ключ
  - Выберите все окружения → Save

- Найдите `SUPABASE_SERVICE_ROLE_KEY`
  - Edit → вставьте **service_role** ключ (ДРУГОЙ!)
  - Выберите все окружения → Save

### 3. Redeploy

**Vercel → Deployments** → последний деплой → ⋯ → **Redeploy**

Дождитесь 2-3 минуты.

### 4. Проверьте сайт ✅

Откройте ваш сайт - ошибка 500 должна исчезнуть!

## 🆘 Если не помогло

Смотрите логи в Vercel:
**Deployments → Runtime Logs**

Или в терминале:
```bash
vercel logs --follow
```

Ищите ошибки:
- "Invalid JWT" - ключ скопирован неполностью
- "Missing environment variables" - не выбрано Production
- "Connection refused" - проблема с URL
