# ⚡ РЕШЕНИЕ ОШИБКИ 500 (одна страница)

## Проблема
Ваши ключи `SUPABASE_SERVICE_ROLE_KEY` и `NEXT_PUBLIC_SUPABASE_ANON_KEY` одинаковые - это неправильно!

## Решение (3 минуты)

### 1. Найдите правильные ключи

**Supabase → Settings (⚙️) → API → Project API keys**

Вы увидите ДВА ключа:

```
✅ anon public
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOi...
[Copy] ← Скопируйте это в блокнот (назовите "ANON")

✅ service_role                    [Reveal] ← Нажмите!
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOi... (ДРУГОЙ!)
[Copy] ← Скопируйте это в блокнот (назовите "SERVICE")
```

### 2. Обновите на Vercel

**Vercel → Settings → Environment Variables**

#### Для NEXT_PUBLIC_SUPABASE_ANON_KEY:
1. Три точки (⋯) → Edit
2. Вставьте **ANON** ключ
3. Окружения: ✅ Production ✅ Preview ✅ Development
4. Save

#### Для SUPABASE_SERVICE_ROLE_KEY:
1. Три точки (⋯) → Edit
2. Вставьте **SERVICE** ключ (ДРУГОЙ!)
3. Окружения: ✅ Production ✅ Preview ✅ Development
4. Save

### 3. Redeploy

**Vercel → Deployments** → последний деплой → ⋯ → **Redeploy**

Дождитесь завершения (2-3 минуты).

### 4. Проверьте

Откройте ваш сайт - ошибка 500 должна исчезнуть! ✅

---

## Если не помогло

### Проверьте логи:

```bash
vercel logs --follow
```

Или: **Vercel → Deployments → Runtime Logs**

### Частые ошибки:

| Ошибка в логах | Причина | Решение |
|----------------|---------|---------|
| "Invalid JWT" | Ключ скопирован неполностью | Удалите и добавьте заново |
| "Missing environment variables" | Не выбрано Production окружение | Редактируйте и выберите Production |
| "Connection refused" | Неправильный URL | Проверьте NEXT_PUBLIC_SUPABASE_URL |
