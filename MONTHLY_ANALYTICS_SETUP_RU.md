# 🎯 Настройка Месячной Аналитики с Perplexity - Быстрый Старт

## ✅ Что было реализовано

Система автоматической месячной аналитики настроения с интеграцией Perplexity AI:

- ✨ **Накопительный анализ**: данные собираются каждую неделю с начала месяца
- 📊 **Структурированные отчеты**: 5 секций анализа (впечатление, позитив, причины спада, рекомендации, размышления)
- 🤖 **Улучшенный промпт**: профессиональный психологический анализ без оценок и суждений
- 🔄 **Автоматизация**: ежедневный cron job проверяет и генерирует аналитику
- 💾 **История**: финальная аналитика месяца сохраняется навсегда

## 🚀 Шаги для запуска

### 1. Применить миграцию базы данных

Откройте Supabase Dashboard → SQL Editor и выполните:

```bash
# Файл: supabase/migrations/024_create_monthly_analytics.sql
```

```bash

```

### 2. Проверить переменные окружения

Убедитесь, что в `.env.local` и в Vercel есть:

```bash

```

### 3. Деплой на Vercel

```bash
git add .
git commit -m "feat: add monthly analytics with Perplexity AI"
git push origin main
```

Vercel автоматически:
- Применит новый cron job (ежедневно в 22:00 UTC)
- Создаст новые API endpoints
- Обновит UI календаря

## 📱 Как это работает для пользователей

### Автоматический режим

1. **Каждые 7 дней** система автоматически:
   - Собирает все записи месяца до текущего дня
   - Отправляет в Perplexity для анализа
   - Сохраняет аналитику в базу данных

2. **В последний день месяца** создается финальная аналитика:
   - Помечается специальным флагом
   - Остается навсегда для исторического просмотра
   - Следующий месяц начинается с нуля

### Ручной режим

Пользователи могут:
- Нажать **"Сгенерировать"** для создания первой аналитики
- Нажать **"Обновить"** для незавершенных месяцев
- Переключаться между месяцами и видеть историческую аналитику

## 🔧 Настройка cron расписания (опционально)

В `vercel.json` можно изменить расписание:

```json
{
  "crons": [
    {
      "path": "/api/cron/monthly-analytics",
      "schedule": "0 22 * * *"  // Каждый день в 22:00 UTC
    }
  ]
}
```

Примеры расписаний:
- `0 22 * * *` - Каждый день в 22:00 UTC
- `0 */6 * * *` - Каждые 6 часов
- `0 0 * * 0` - Каждое воскресенье в полночь

## 🧪 Тестирование

### Локальное тестирование

```bash
# 1. Запустить dev сервер
npm run dev

# 2. Создать несколько записей в дневнике

# 3. Вручную сгенерировать аналитику
curl -X POST http://localhost:3000/api/analytics/generate \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{"year": 2024, "month": 11}'

# 4. Проверить на странице календаря
open http://localhost:3000/calendar
```

### Тестирование cron job

```bash
# Запустить вручную с секретом
curl -X GET http://localhost:3000/api/cron/monthly-analytics \
  -H "Authorization: Bearer YOUR_CRON_SECRET"
```

## ⚠️ Важные моменты

### Минимальные требования

- **3+ записей** в месяце для генерации аналитики
- **Активная подписка** (subscription или personal tier)
- **Настроенный PERPLEXITY_API_KEY**

### Когда генерируется аналитика

✅ **Генерируется:**
- Каждые 7 дней месяца (7, 14, 21, 28)
- Последний день месяца (независимо от количества дней)
- При ручном запросе пользователя

❌ **Не генерируется:**
- Если уже существует для этой недели
- Если меньше 3 записей
- Если нет активной подписки

### Структура данных

```typescript
interface MonthlyAnalytics {
  year: number              // 2024
  month: number             // 11 (1-12)
  week_number: number       // 1, 2, 3, 4
  days_analyzed: number     // 7, 14, 21, 30
  
  // Секции анализа
  general_impression: string
  positive_trends: string
  decline_reasons: string
  recommendations: string
  reflection_directions: string
  
  is_final: boolean         // true в последний день месяца
  status: 'completed'
}
```

## 🎉 Готово к использованию!

После деплоя система начнет работать автоматически:

1. ✅ Cron job запустится в 22:00 UTC каждый день
2. ✅ Проверит всех пользователей с активными подписками
3. ✅ Сгенерирует аналитику для тех, кому нужно (каждые 7 дней)
4. ✅ Пользователи увидят аналитику на странице календаря
5. ✅ В конце месяца создастся финальная аналитика

## 🆘 Troubleshooting

**Аналитика не появляется:**
- Проверьте, что у пользователя есть 3+ записей
- Проверьте статус подписки
- Посмотрите логи в Vercel Dashboard

**Cron не работает:**
- Убедитесь, что проект задеплоен на Vercel
- Проверьте Vercel Dashboard → Cron Jobs
- Проверьте переменную CRON_SECRET

**Ошибки Perplexity:**
- Проверьте PERPLEXITY_API_KEY
- Проверьте баланс API ключа
- Посмотрите логи Vercel Functions
