## Основные изменения

### База данных
- Создана таблица monthly_analytics с RLS политиками
- Накопительный принцип анализа (7, 14, 21, 30+ дней)
- Финальная аналитика сохраняется навсегда

### API
- POST /api/analytics/generate - генерация аналитики
- GET /api/analytics - получение аналитики по месяцу
- GET /api/cron/monthly-analytics - автоматический cron job

### Perplexity Integration
- Новый метод analyzeMonthlyMood() с улучшенным промптом
- Структурированный ответ по 5 секциям
- Автоматический парсинг и форматирование

### UI
- Обновлен календарь с блоком "Аналитика и рекомендации"
- Кнопки генерации и обновления аналитики
- Отображение всех секций анализа
- Индикатор финальной аналитики

### Автоматизация
- Cron job запускается ежедневно в 22:00 UTC
- Проверяет всех пользователей с активной подпиской
- Генерирует аналитику каждые 7 дней
- Создает финальную аналитику в последний день месяца

## Механика работы

1. Пользователь заполняет дневник
2. Каждые 7 дней система автоматически:
   - Собирает все записи месяца до текущего дня
   - Отправляет в Perplexity для анализа
   - Сохраняет результат в БД
3. В последний день месяца создается финальная аналитика
4. Пользователь видит аналитику на странице календаря
5. При переходе к другим месяцам отображается историческая аналитика
