# 🔧 Исправление ошибки "User not found" при анализе

## Проблема
При нажатии на кнопку "📊 Анализ" в админ-панели возникает ошибка **"User not found"**.

## Причина
RLS (Row Level Security) политики блокируют доступ администратора к данным пользователей.

## 🔍 Диагностика

Если проблема сохраняется, проверьте следующее:

### 1. Проверка админа в базе данных

Выполните в SQL Editor:

```sql
-- Ваш текущий email (замените на свой)
SELECT 
  u.id,
  u.email,
  u.sso_uid,
  p.role,
  p.active
FROM users u
LEFT JOIN psychologists p ON p.user_id = u.sso_uid
WHERE u.email = 'site4people@gmail.com';
```

**Ожидаемый результат:**
- `role` должна быть `'admin'`
- `active` должна быть `true`

### 2. Если вас нет в таблице psychologists

Выполните (замените данные на свои):

```sql
-- Сначала найдите свой sso_uid
SELECT id, email, sso_uid FROM users WHERE email = 'site4people@gmail.com';

-- Добавьте себя как админа (используйте sso_uid из предыдущего запроса)
INSERT INTO psychologists (user_id, email, full_name, role, active)
VALUES (
  'ВАШ_SSO_UID_ИЗ_ПРЕДЫДУЩЕГО_ЗАПРОСА',
  'site4people@gmail.com',
  'Daniil Chistoforov',
  'admin',
  true
)
ON CONFLICT (email) DO UPDATE SET
  role = 'admin',
  active = true;
```

### 3. Проверка RLS политик

```sql
-- Должны быть политики "Admins can view all users" и "Admins can update all users"
SELECT policyname, cmd, qual
FROM pg_policies
WHERE tablename = 'users'
ORDER BY policyname;
```

## 📝 Что делает исправление

1. **Обновляет RLS политики для `users`**
   - Разрешает админам видеть всех пользователей
   - Разрешает админам обновлять пользователей

2. **Обновляет RLS политики для `monthly_analytics`**
   - Разрешает админам создавать аналитику для любого пользователя
   - Разрешает админам видеть всю аналитику

## ❓ Частые вопросы

**Q: После применения скрипта все равно ошибка**
A: Проверьте в консоли браузера (F12 → Network), что запрос `/api/admin/analytics/manual` теперь возвращает другую ошибку. Возможно, проблема уже не в доступе к users.

**Q: Показывает 404 на API endpoint**
A: Убедитесь, что приложение задеплоено на Vercel и все изменения запушены в Git.

**Q: Как узнать свой sso_uid?**
A: Выполните `SELECT id, email, sso_uid FROM users WHERE email = 'ваш@email.com';`
