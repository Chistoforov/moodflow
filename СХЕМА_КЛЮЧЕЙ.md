```
                    ║

```

## 🔍 Как проверить визуально

Если скопировать оба ключа в текстовый редактор:

```
anon:         eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJzdXBhYmFzZS... (~300 символов)
service_role: eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJzdXBhYmFzZS... (~400 символов)
              └────────────── одинаковое начало ─────────────┘
                                                    └── но РАЗНОЕ продолжение! ──┘
```

Оба начинаются одинаково (`eyJhbG...`), но:
- Разная длина
- Разное содержание в середине и конце
- service_role обычно длиннее

## 🆘 Если все еще не работает

Проверьте логи:

```bash
# Через CLI
vercel logs --follow
```

Или в браузере:
Vercel → Deployments → последний деплой → Runtime Logs

Ищите ошибки:
- "Invalid JWT" → ключ скопирован неправильно
- "Missing environment variables" → переменные не для Production
- "Connection refused" → проблема с URL
